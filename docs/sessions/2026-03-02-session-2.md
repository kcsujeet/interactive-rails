# Session: 2026-03-02 - Session 2

## Summary

Implemented interactive observe and reward phases for the level system, using L12 (Authorization) as the reference implementation. Previously, observe and reward phases were 100% passive (auto-incrementing counters, static pipeline). Now players have agency in both phases.

## Changes Made

### New Hooks (2 files)
- **`frontend/src/hooks/useDiscoveryGating.ts`**: Tracks what the player has discovered during the observe phase. Config accepts `DiscoveryDef[]` with `minRequired` option. API: `discover(id)`, `isDiscovered(id)`, `isUnlocked`. No wrong attempts, no auto-advance. Pure exploration gating.
- **`frontend/src/hooks/useStressTest.ts`**: Manages reward phase stress-test mechanics. Config accepts `StressScenario[]`. API: `fireRequest(scenarioId)`, `toggleAutoFire()`, `reset()`. Auto-fire gates behind 3+ manual fires with escalating speed (2000ms down to 500ms).

Responses reveal vulnerabilities. Amber-themed buttons and `>` prompt distinguish it from SimulatedTerminal's `$` prompt. Each probe fires once.

## Architecture Decisions

- Discovery gating mirrors `useStepGating` in shape but is simpler (no wrong attempts, no star rating). This keeps the two concerns cleanly separated.
- ProbeTerminal uses amber theming and `>` prompt to visually distinguish from SimulatedTerminal's green `$` prompt. This signals "exploration mode" vs "build mode".
- Stage inspector data is defined as a map keyed by stage ID, with a separate `STAGE_DISCOVERY_MAP` linking stage clicks to discoveries. This keeps the mapping explicit and auditable.
- StressTestPanel lives in the center panel below the pipeline (matching ProbeTerminal's position in observe phase). Left panel shows legend + counters for quick glance.
- PipelineFlow `'danger'` variant provides red counterpart to `'active'` (green). Both observe and reward phases use it for reactive node coloring.
- Pipeline stages are built with `useMemo` reacting to probe/stress-test state so nodes update immediately on each interaction.
