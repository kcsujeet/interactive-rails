## Session 2: L58 deep pass (The Architect capstone redesign)

### Design decisions (design-level worksheet outputs)

- **Narrative (Step 1).** Problem in one sentence: billing keeps hurting everyone else's customers because it shares the monolith's runtime, database, and deploy unit despite clean package boundaries. Actor table: Customer (feels latency/outages), Billing team (hotfix, month-end batch), Storefront monolith (packs), Shared PostgreSQL, Deploy pipeline; reward adds Billing Service, Billing DB, Gateway, Event bus, Flag gate. The old before-state (sync side effects, "no state machine") erased L54/L56's work and is gone.
- **The honest remaining pains** (post-L56 world): (1) deploy coupling: a one-line fee fix waits ~2h behind the full pipeline while customers stay overcharged; (2) shared-DB contention: month-end invoicing (2M orders) drags checkout p95 180ms -> 2.1s; (3) shared-runtime blast radius: a billing PDF-renderer leak OOM-kills the shared Puma workers and 503s the storefront.
- **Smell test (Step 0.45, written):** the player fires "Ship a one-line billing hotfix" and watches customers stay overcharged for two hours because the fix is stuck behind 47 unrelated commits. Reaction: "billing keeps hurting everything around it; it needs its own everything." PASS: damage is customer-and-operator concrete.
- **Probe playthrough:** (1) hotfix: billing pack FIX badge -> pipeline amber "building the WHOLE monolith / 47 commits" -> customer red "overcharged for 2 more hours". (2) invoicing: billing batch writes flood the shared primary (94% badge) -> checkout queries queue -> customer red "checkout crawling / 2.1s". (3) OOM: billing LEAK -> runtime edge "same workers, same fate" -> storefront OOM -> customer 503. Three visually distinct failure shapes (deploy path, database path, runtime path), test-enforced.
- **Canonical purpose (Step 0.5, doc-verified):** Fowler's Strangler Fig (martinfowler.com/bliki/StranglerFigApplication.html) quoted for why incremental beats big-bang ("replacements seem easy to specify, but often it's hard to figure out the details of existing behavior"; transitional architecture's "reduced risk and earlier value ... outweigh its costs"). Cited in learningContent and used as the flaw in the big-bang wrong options.
