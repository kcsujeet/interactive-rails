# Session: 2026-02-24 #3

## Goal
Fix 7 structural/narrative issues identified in the Acts 1-2 audit.

## Issues Fixed

### 1. L7 Serializers: password_digest reference removed
**Problem:** L7 trigger said "including password_digest" but no User model exists until L9. The codeExample showed a password_digest field on a Post response.
**Fix:** Changed trigger to "internal timestamps and every column." Updated codeExample to show `lock_version` and raw timestamps instead of password_digest. Updated learningContent goal, conceptExplanation, and commonMistakes to reference "timestamps, lock columns, internal IDs" instead.
**File:** `frontend/src/features/act1-foundation/content.ts`

### 2. L14 Strong Params: params.expect overlap with L6 resolved
**Problem:** L6 teaches `params.expect`, then L14's scenario said "controller passes raw params straight to the model" and "uses params[:post] directly", contradicting L6.
**Fix:** Reframed L14 as a security audit of an existing (but too broad) whitelist, not introducing params.expect from scratch. New scenario: "Your controller uses params.expect, but the whitelist is too broad. A malicious user sends admin: true." Updated trigger, observation, rootCause, codeExample, goal, learningContent, commonMistakes, and hint.
**File:** `frontend/src/features/act2-users-security/content.ts`

### 3. L11-L12 narrative bridge added
**Problem:** L11 Callbacks ends with email normalization. L12 Authorization jumps to "User A can edit User B's posts" with no connection to prior levels.
**Fix:** Updated L12 trigger to: "Users can authenticate, data is validated, and emails are normalized. But User A can still edit User B's posts."
**File:** `frontend/src/features/act2-users-security/content.ts`

### 4. L15 Scopes & Enums: complexity regression reframed
**Problem:** Scopes & Enums (model-layer DSL) felt like a step back after Auth (L12), Testing (L13), and Strong Params (L14).
**Fix:** Reframed as the visibility complement to authorization. New trigger: "Authorization controls who can act, but the API still returns ALL posts. You control actions, now control visibility." Updated observation to reference Pundit policies. Updated rootCause to reference L12. Updated learningContent goal to emphasize "completing the security arc."
**File:** `frontend/src/features/act2-users-security/content.ts`

### 5. L15 missing Post.status schema addressed
**Problem:** L15 referenced a Post.status column but no prior level added it via migration.
**Fix:** Updated codeExample to show the migration (`AddStatusToPosts`) as the first step, including the integer column with default and index. Updated goal to include "Add a status column."
**File:** `frontend/src/features/act2-users-security/content.ts`

### 6. spec.md Act 1 table updated to match code
**Problem:** Spec had L1=First Boot through L8=Seeds, but code has L1=Environment, L2=First Boot through L8=Associations (no Seeds level).
**Fix:** Updated Act 1 table to add Environment as L1, shifted other levels to match code numbering (L2=First Boot, L6=Controller, L7=Serializers, L8=Associations). Removed Seeds row. Updated Rails 8 Features Integration table: `rails new --api` and SQLite to L2, `params.expect()` to L6/L14, `enum` to L15.
**File:** `docs/spec.md`

### 7. L12 Authorization trigger improved
The L12 trigger now explicitly references the progression from L9 (auth), L10 (validations), and L11 (callbacks) before identifying the authorization gap.

## Narrative Arc After Fixes

Act 2 now reads as a coherent security progression:
- L9 Auth: Who are you?
- L10 Validations: Is the data valid?
- L11 Callbacks: Is the data clean?
- L12 Authorization: Are you allowed to do this? (bridges from L9-L11)
- L13 Testing: Prove it all works
- L14 Strong Params: Audit your param whitelist (builds on L6's params.expect)
- L15 Scopes & Enums: Control what users SEE (visibility complement to L12's action control)
- L16 CORS: Connect the React frontend
