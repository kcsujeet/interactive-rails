## Fix: Green dot flashing at top-left during edge animations

### What was done
- The animated dots on edge connectors used positive `begin` offsets (`"0s"`, `"0.5s"`, `"1s"`) to stagger 3 dots. SVG `<circle>` elements render at coordinate (0,0) until their `animateMotion` begins, causing dots with `begin: "0.5s"` and `begin: "1s"` to flash at the top-left corner of the React Flow canvas before starting their path animation.
- Fixed by using negative begin offsets (`"0s"`, `"-0.5s"`, `"-1s"`) instead. With `repeatCount="indefinite"`, a negative begin means the dot starts already partway through its cycle, achieving the same visual stagger without any pre-animation delay. This matches the pattern already used by `PipelineFlow` and `QueryZoneFlow`.
- File changed: `Level35ActiveStorage.tsx` line 1577
- All four checks pass.
