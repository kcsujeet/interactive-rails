## FlowDiagram: Reusable React Flow Wrapper

### What was done
- Created `FlowDiagram` component (`frontend/src/components/levels/FlowDiagram.tsx`) as a reusable wrapper around React Flow that handles all shared boilerplate (fitView, proOptions, nodesConnectable, nodesFocusable, elementsSelectable, onNodeClick translation)
- Extracted `FlowHandles` component: renders 8 invisible handles (all 4 positions, source + target) that every custom node needs
- Extracted `AnimatedDots` component: renders SVG `<circle>` + `<animateMotion>` dots along a path, used by all edge types
- Exported `DotConfig` type and `reversePath` utility for reverse-direction dot animation
- Refactored `PipelineFlow` to use `FlowDiagram`, `FlowHandles`, and `AnimatedDots` internally (zero breaking changes, all 13 consuming levels work unchanged)
- Refactored `QueryZoneFlow` to use `FlowDiagram`, `FlowHandles`, and `AnimatedDots` internally (zero breaking changes)
- Refactored Level 35 Active Storage to use `FlowDiagram`, `FlowHandles`, `AnimatedDots`, and `reversePath` (removed raw ReactFlow setup)
- Net result: -106 lines of code, 3 copies of boilerplate eliminated, future levels can use `FlowDiagram` directly with custom node/edge types

### Design decisions
- Thin wrapper + composable building blocks (not a mega-component). Node and edge types remain level-specific because they render fundamentally different content
- `PipelineFlow` and `QueryZoneFlow` remain as convenience wrappers with unchanged public APIs
- Future levels that need custom visualizations import `FlowDiagram` + building blocks directly
