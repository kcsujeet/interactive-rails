## Fix: Green dot flashing at top-left during edge animations

### What was done
- The animated dots on edge connectors used positive `begin` offsets (`"0s"`, `"0.5s"`, `"1s"`) to stagger 3 dots. SVG `<circle>` elements render at coordinate (0,0) until their `animateMotion` begins, causing dots with `begin: "0.5s"` and `begin: "1s"` to flash at the top-left corner of the React Flow canvas before starting their path animation.
- Fixed by using negative begin offsets (`"0s"`, `"-0.5s"`, `"-1s"`) instead. With `repeatCount="indefinite"`, a negative begin means the dot starts already partway through its cycle, achieving the same visual stagger without any pre-animation delay. This matches the pattern already used by `PipelineFlow` and `QueryZoneFlow`.
- File changed: `Level35ActiveStorage.tsx` line 1577
- All four checks pass.

## Fix: Dot animation direction not reversing on connectors

### What was done
Edge connector dots always flowed from client to server even during response animations. The root cause: dot IDs were `${id}-d${i}` which didn't include the `reverse` flag. When `reverse` flipped from `false` to `true`, React reused the same `<circle>` DOM elements (same keys), and the running SVG `<animateMotion>` ignored the changed `path` prop.

Fix: include direction in dot IDs (`${id}-${dir}-d${i}` where `dir` is `'fwd'` or `'rev'`). When direction changes, React unmounts old circles and mounts new ones with the reversed path, forcing the SVG animation to restart in the correct direction.

- File changed: `Level35ActiveStorage.tsx` (UploadEdge dot ID generation)
- This fix alone was insufficient (see next entry)

## Fix: reversePath regex didn't match getStraightPath output format

### What was done
The real root cause of dots always flowing client-to-server: `reversePath()` in `FlowDiagram.tsx` expected straight-line paths in `M x y L x y` format (space-separated, space before L), but React Flow's `getStraightPath()` returns `M x,yL x,y` (comma-separated, no space before L). The regex never matched, so `reversePath` returned the path unchanged and dots always went in the default direction.

Fixed the regex from `/^M\s+([\d.-]+)\s+([\d.-]+)\s+L\s+([\d.-]+)\s+([\d.-]+)$/` to `/^M\s*([\d.-]+)[,\s]+([\d.-]+)\s*L\s*([\d.-]+)[,\s]+([\d.-]+)$/`, which handles both formats. Output also uses comma format to match React Flow's convention.

- File changed: `FlowDiagram.tsx` (reversePath regex)
- All four checks pass, all 1050 tests pass
- This affects all levels using `reversePath` with straight-line edges (L35 and any future levels)
