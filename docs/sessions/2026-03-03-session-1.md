- **How It Works**: Replaced 4 generic cards with 3 phase cards (Observe, Build, Stress Test) matching actual gameplay. Added numbered badges, chevron arrows between phases, colored subtitles.

### New component (`frontend/src/components/landing/HeroPipeline.tsx`)
- Wrapper around `PipelineFlow` with 5 demo stages (Request, Router, Controller, Model, Database) and animated dot connections (`clean` for first two edges, `mixed` for last two). Database node uses `danger` variant with "N+1!" badge.

## Architecture Decisions

- Used `client:visible` for HeroPipeline hydration so it only loads when scrolled into view, keeping initial page load light.
- Placed HeroPipeline in `components/landing/` since it's a landing-page-specific presentational component, not a game feature.
