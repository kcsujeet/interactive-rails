## Architecture Decisions

- **Static HTML pipeline over React Flow**: The hero pipeline was initially built using the PipelineFlow React component with `client:visible` hydration. After several iterations (horizontal looked cramped, vertical required modifying a shared component), switched to pure static HTML with CSS animations. Benefits: no React hydration needed, full layout control, no modifications to shared game components.
- **Badge over raw spans**: Replaced all 26 raw `<span>` code pills with `<Badge>` component to follow the "never use raw HTML when a design system component exists" rule.
- **Spacing matched to existing patterns**: How It Works cards were iteratively refined to match the concepts section spacing (p-6, w-12 h-12 icon containers, text-lg headings) rather than inventing new values.

## Lessons Learned

- Don't modify shared game components (PipelineFlow) for landing page one-off needs. Build a separate static version instead.
- When setting spacing for new sections, find an existing section that looks right and match those exact values rather than guessing.
- Check all instances when fixing copy (e.g., "Rails 8" appeared in both hero and footer).
- **Always verify dark mode is active.** The `.dark` class must be on the `<html>` element for the design system to work. Without it, all semantic tokens (bg-background, text-foreground, etc.) resolve to light-mode values, making the page look completely wrong.
- Modern dark-theme patterns: use semi-transparent white borders (`border-white/[0.08]`) instead of solid zinc borders, diffuse shadows (`0_8px_40px`), and subtle radial gradient glows for depth.
