## Session 1: Spec reconciliation + spec-to-registry consistency test

### Context

Discussed overall project direction. Key decision recorded to memory: this project is a personal learning tool (the goal is expert-level Rails for the author), not a product to ship. Agreed priority order for improvements: (1) spec reconciliation + consistency test, (2) executable validators (answer-leak linter, structural checks), (3) Playwright smoke tests per interaction archetype, (4) free-input terminal mode, (5) myapp homework bridge and act-end boss levels.

### What was changed

1. **`src/lib/__tests__/spec-consistency.test.ts` (new, TDD-first).** Parses `docs/spec.md` and pins it to the live curriculum in `src/lib/acts-registry.ts`:
   - act count, act names, and per-act level ranges from headings (`## ACT N: Name (Levels A-B)`)
   - every level number and name in every act table, exact and in order
   - the headline stats line (`**58 levels, 7 acts**`) and intro (`58-level`)
   - the Rails 8 Features Integration table: each feature row's level references are computed from the registry via a human-verified feature-to-level-name map, so renumbering can never silently invalidate the table
   - Written first and run against the old spec: 22 of 31 tests failed, confirming the drift detection works. All 31 pass after the rewrite.

2. **`docs/spec.md` (rewritten).** The old spec had drifted badly from the shipped curriculum:
   - claimed a "55-level progression" in the intro while stats said 58
   - Act 2 was titled "Guards & Gates" with 7 levels including CORS at L15; the real Act 2 is "Users & Security" with 6 levels, CORS lives in Act 6 as L41
   - Encrypted Attributes was listed at L36 (Act 5); it is actually L10 (Act 2)
   - Callbacks & Normalizations was listed at L11; it is actually L15 (Act 3, matching the current level-15-callbacks work)
   - Act 1 ordering was wrong (spec: CRUD L4, Routes L5, Controller L6, Associations L8; real: Associations L4, CRUD L5, Routes L6)
   - Action Mailer / Background Jobs were in Act 3; they are actually L35/L36 in Act 5
   - the Rails 8 feature table had `rate_limit` at L43 (real: L42) and Kamal at L42 (real: L49), plus stale numbers for `params.expect` (L14 vs real L13), `normalizes` (L11 vs L15), `generates_token_for` (L21 vs L35), Solid Queue (L22/L46 vs L36/L44), Solid Cache (L30 vs L28), `encrypts` (L36 vs L10), `Current` (L12 vs L11)
   - All tables rewritten from the registry. Scenario column condensed from each level's actual `trigger.description` (dumped from the registry, not paraphrased from memory). The stale per-act "Tests?" columns were dropped. Added a "Source of truth" note pointing at the registry and the new test. Noted that `act8-mastery` is an empty stub directory, not part of the live curriculum.

### Why

`docs/spec.md` is the document CLAUDE.md and the audit skills instruct every session to cross-reference for level scenarios and concepts. A drifted spec silently corrupts every audit that trusts it. Since the project's purpose is the author's own learning, content accuracy is the top priority; this closes the largest known accuracy hole and makes the whole class of drift a failing test instead of a human catch.

## Session 2: Executable validators + free-input terminal + Playwright smoke + homework bridge

Four workstreams executed in one batch (user approval: "do all in one go, review at end").

### 1. Curriculum-wide validators (executable rules)

- **`src/lib/testing/level-source-analysis.ts` (new, TDD-first, 16 unit tests).** Parses level source with the TypeScript compiler API (most levels keep choice data as unexported module-scope consts, so imports cannot introspect them). Extracts option groups (arrays of objects with a `correct` flag), PipelineFlow JSX tags, and PROBE_DISCOVERY_MAP literals. Decision-modal `consequence` counts as the feedback surface.
- **`src/lib/__tests__/curriculum-quality.test.ts` (new sweep, 5 tests).** Hard-enforces across all 58 levels: exactly one correct option per group, correct never first in source order, every wrong option has feedback, no mixed option colors, feedback never quotes the correct answer verbatim, no discovery unlocked by two probes, every PipelineFlow tag passes activeConnections explicitly, no em dashes anywhere in src.
- **`scripts/report-answer-leaks.ts` + `bun run report:leaks` (new).** The token-level leak heuristic produces 160 candidates, mostly generic-Rails-vocabulary noise, so it ships as a report for the manual audit pass instead of a CI gate. The verbatim phrase-quote variant IS gated.
- **Violations found and fixed:**
  - L7 (Controller): PROBE_DISCOVERY_MAP was many-to-many; the observe gate could clear without firing the DELETE probe. Added a fifth discovery ("All five REST actions are dead") and made the map 1:1 (probes: get-fails / post-fails / all-actions-dead; stage clicks: routes-work / no-controller).
  - Correct-answer-first reorders: L31 soft-deletes (2 arrays), L54 state machines (1 array).
  - 6 real answer leaks rephrased (components + mirrored tests): L9 feedback named the built-in "authentication generator"; L21 named pg_query; L26 named OPTIONS/:limit and the pagy() call; L28 named db:prepare; L55 named bin/packwerk and init.
  - 69 em dashes purged across 25 src files (comma/colon replacements; describe() titles use colons).
  - PipelineFlow activeConnections wired in all 7 offending files: L20/L29/L35/L36 observe+reward now burst all edges on probe/scenario fire (probeTick / stressTest.results.length as animationTick) and stay dormant before; L49 observe/build pass explicit [] (their edges have no dots, so behavior-identical); L49 reward bursts on fire. Emptied the KNOWN_AUTO_ANIMATING_EDGES baseline in scripts/__tests__/level-reveal-consistency.test.ts.

### 2. Free-input terminal mode (recall over recognition)

- **`src/components/levels/terminal-input-matching.ts` (new, TDD-first, 8 tests).** Case-sensitive, whitespace-normalized matching of typed input against each command's `command` or `label`. `FREE_INPUT_MISS_LIMIT = 2`.
- **`SimulatedTerminal.tsx` reworked.** The prompt line is now a real input: the player types the command (recall). Typing a wrong option's command shows its feedback (counts as a wrong answer). Unrecognized input echoes a shell-style error and counts as a miss but NOT a wrong answer (a typo is not a choice). After 2 misses the option buttons appear as the recognition fallback. Uniform across every terminal level via TerminalChoiceStep (no per-level opt-outs, per the consistency rule).
