# Session: 2026-02-21 - Session 1

## Summary

## Changes

### 1. Resizable panel layout (`54aef03`)

**What:** Replaced fixed-width left/right panels with draggable resizable panels using `react-resizable-panels` v4.6.5.

**Why:** The 3-panel level layout (Instructions | Canvas | Code Preview) had hardcoded widths (`w-64`, `w-80`). Users couldn't adjust panel sizes to their preference.

**How:** `LevelLayout` renders a `Group` wrapper. `LeftPanel`, `CenterPanel`, `RightPanel` each wrap their content in a `Panel` with percentage-based sizing and emit `Separator` drag handles.

**Key learnings:**
- In `react-resizable-panels` v4, numeric size values are **pixels**, not percentages. Must use strings like `"18%"`.
- `Panel` and `Separator` must be direct **DOM** children of `Group`. Since React fragments don't create DOM nodes, returning `<><Panel/><Separator/></>` from wrapper components works fine.

**Files:** `frontend/package.json`, `frontend/src/components/levels/LevelLayout.tsx`

### 2. Levels button navigation fix (`cba6d00`)

**What:** Changed the "Levels" button in `LevelHeader` from an `onClick` handler to a proper `<Button asChild><a href="/acts/{actNumber}">` link.

**Why:** The button was using `onExit` callback which navigated to the wrong URL (level briefing page instead of levels list). A navigation link should be a link, not a JS click handler.

**How:** Replaced with `<Button asChild variant="link">` wrapping an `<a>` tag. Removed the now-unused `onExit` prop from `LevelHeader`, `LevelComponentProps`, and all 50 level components.

**New CLAUDE.md rule added:** "Never use raw HTML elements when a design system component exists."

**Files:** `LevelHeader.tsx`, `levels-registry.ts`, all 50 level components
