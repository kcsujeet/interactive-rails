## Session 3: L57/L58 world reconciliation (gateway fronts the monolith)

### The design decision

L57 presumed billing/analytics running as separate microservices on ports 3001-3003 with three auth schemes (JWT, API key, Basic), none of which was ever built, and all of which contradicted L58's premise that billing has never been extracted. Chosen fix: the gateway fronts the MONOLITH. The honest story: the mobile app stitches the dashboard from six sequential calls to the same Rails app (2.4s on 4G, 480KB payload for a 6KB screen, no per-section fallback, six paths baked into shipped binaries). The gateway is a controller in the app: authenticate once (L9's concern), map sections to the L55 packages' public readers (in-process method calls, not HTTP-to-self), shape one payload for the screen, rescue per section (reporting via L46's Rails.error.report), one rate_limit at the door (L42). The route-moved probe/scenario pair explicitly teaches the seam: shipped apps break when internal paths move; behind the gateway the backend reorganizes freely, which is exactly what makes L58's billing extraction invisible to clients. L58's recap line updated to credit the gateway as that seam (and to stop crediting it with circuit breakers, which are L38's lesson for external calls).

### What changed

- **Component data block rewritten** (defaults, discoveries, probes, observe/reward frames, inspector map, scenarios, all six build steps, getCodeFiles): no ports, no JWT/Faraday/Stoplight/threads (all untaught), no phantom services. Observe nodes are endpoint groups of the same app; reward nodes are the packages with in-process edges. Numbers kept honest: 1 round trip ~400ms (was 6 x ~400ms), no "150ms" sub-RTT claims.
- **Probes made verb-led with distinct damage**: 4G latency cascade, metered-connection overfetch (480KB/6KB), analytics-500 blanking the screen, route-rename 404ing shipped app versions. The two inspection probes ("Inspect auth mechanisms", "cat services.yml") are gone.
- **Scenarios**: 4 paired 1:1 with probes (same ids/labels) + 1 reward-only rate-limit burst; every scenario has frames (the 3 dead duplicate buttons are gone) and a 3-6 bullet "same customer, but..." story.
- **Audit leak fixes landed with the rewrite**: middleware feedback no longer names "controller"; fetch_or_default/rate_limit no longer named in feedback; content goal describes outcomes, not tools.
- **Content file rewritten** (trigger/observation/rootCause/codeExample/goal/learningContent/hint) for the BFF-gateway-as-seam story, including the "when a package later becomes its own service, clients never notice" bridge to the capstone.
- **New test file** `act7-scale/__tests__/Level57APIGateway.test.ts` (16 tests): pedagogy helpers (1:1 map, probe/scenario pairing, stories, step quality, scenario basics), frames coverage both directions (no dead buttons, no orphan frames), observe-never-touches-gateway (show-only-what-exists), distinct animations per probe, and a world-coherence check that greps the level for the forbidden phantom topology (:3001-:3003, users.internal).
- Cross-reference pins added for the new intentional L9/L46/L55 references.

### Audit-skill mandatory outputs

- **Smell test (written)**: a first-time developer fires "Open the dashboard on 4G" and watches six calls crawl one at a time into "2400ms of spinner". Reaction: "the app makes six trips for one screen, that is obviously slow, I must fix it." PASS: damage is the customer's spinner/bill/blank screen, labels are plain English.
- **Probe playthrough**: (1) latency: edges pulse A->B->C sequentially with round-trip labels, client counts calls 1/6..5/6, ends red "2400ms of spinner". (2) overfetch: reverse dots (payload flowing down), KB badges per node, ends "480KB downloaded / screen shows ~6KB". (3) section-down: one green node, one red 500 node, client blank-screen red. (4) route-moved: no successful flow at all; server node RENAMED -> 404 badge -> client "until an app-store release ships". Four visually distinct failure modes (also enforced by test).
