## Batch 24: L49 Feature Flags scaffolding (Batch A)

### Renumber cascade

Inserting a new level into the middle of the curriculum requires
shifting every downstream level by +1. The cascade:

- L49 Multi-Database -> L50
- L50 State Machines -> L51
- L51 Multi-Tenancy -> L52
- L52 Observability -> L53
- L53 Modular Monolith -> L54
- L54 Domain Events -> L55
- L55 API Gateway -> L56
- L56 Sharding -> L57
- L57 The Architect (capstone) -> L58

For each level, five things rename:

- The component directory (`level-XX-slug` -> `level-(XX+1)-slug`).
- The component file inside it (`LevelXXSuffix.tsx` ->
  `Level(XX+1)Suffix.tsx`).
- The content file (`level-XX-slug.ts` -> `level-(XX+1)-slug.ts`).
- The class name in the component file (`LevelXXSuffix` ->
  `Level(XX+1)Suffix`).
- The exported variable in the content file (`levelXXSuffix` ->
  `level(XX+1)Suffix`).
- The `levelNumber:` and `id:` fields inside the content file.

Plus references to all of those in `act7/content/act.ts`,
`act8/content/act.ts`, and `src/lib/levels-registry.ts`.

Done with a single `perl -i -pe` script working highest-to-lowest
(L57 renamed first to free up the L57 namespace for the L56-to-L57
rename, etc.). The script renames file paths via `git mv` and
substitutes patterns across all `.ts` and `.tsx` files in `src/`.

One bug found and fixed: my first pass lowercased the suffix's
first letter when computing the variable form, but the actual
variable convention is `level<NN>PascalCase` (no leading-letter
lowercase). Fixed with a second targeted `perl -i -pe` pass.

The curriculum-overview comment block at the top of
`levels-registry.ts` updated to reflect the new ranges (Act 6
L41-L49 now 9 levels; Act 7 L50-L55; Act 8 L56-L58).
