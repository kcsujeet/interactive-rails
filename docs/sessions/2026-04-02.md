## Better Auth Integration

### What was done
- Migrated from custom JWT-based authentication to Better Auth
- **Worker (backend):**
  - Installed `better-auth` package
  - Created `src/lib/auth.ts` with factory function pattern (creates auth instance per-request since D1 bindings are only available in request context)
  - Mounted Better Auth handler at `/api/auth/**` in `src/index.ts`, replacing old custom auth routes
  - Updated auth middleware (`src/middleware/auth.ts`) to use Better Auth's `getSession()` API
  - Added `BETTER_AUTH_SECRET` and `BETTER_AUTH_URL` env vars to `wrangler.toml` and `types.ts`
  - Created D1 migration SQL (`src/db/migration-better-auth.sql`) with Better Auth tables: user, session, account, verification
  - Configured `username` as an additional field on the user model (unique, required, input)
  - Configured password reset handler (console.log for now, to be replaced with real email provider)
  - Set trusted origins for CORS

- **Frontend:**
  - Installed `better-auth` package
  - Created `src/lib/auth-client.ts` with React client and `inferAdditionalFields` plugin for typed `username` field
  - Updated `LoginForm.tsx` to use `authClient.signIn.email()`
  - Updated `SignupForm.tsx` to use `authClient.signUp.email()` with `username` field
  - Updated `authStore.ts` to verify sessions via Better Auth client
  - Removed old auth functions (`login`, `signup`, `getMe`) from `api.ts`

### Key decisions
- Used Better Auth's native D1Database support (no Drizzle ORM needed)
- Factory function pattern for auth instance creation (Cloudflare Workers require per-request D1 access)
- `inferAdditionalFields` plugin on client to type the custom `username` field across the worker/frontend boundary
