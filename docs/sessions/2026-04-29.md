### L48 Deployment: Kamal-related fixes

- Removed fabricated `boot.pre-connect:` syntax for migrations.
  Replaced with two real patterns: container-boot migrations OR
  `kamal app exec --reuse 'bin/rails db:migrate'`. Cited the
  [Kamal commands docs](https://kamal-deploy.org/docs/commands/running-commands-on-servers/)
  for `--reuse` semantics. Removed the unverified "pre-deploy is
  the conventional place for migrations" claim because the
  [hooks overview](https://kamal-deploy.org/docs/hooks/overview/)
  describes pre-deploy as "final checks before deploying, e.g.,
  checking CI completed."
- Removed fabricated healthcheck syntax (`path:` + `cmd:` together).
  Verified against [proxy.rb source](https://github.com/basecamp/kamal/blob/main/lib/kamal/configuration/proxy.rb)
  that the proxy healthcheck only supports `path`, `interval`,
  `timeout`. Replaced with a `.kamal/hooks/post-deploy` example
  that does the deeper smoke test via `kamal app exec --reuse`.

### L37 Real-Time: Rails 8 auth generator fix

- Replaced fabricated `User.authenticate_session_token(token)`
  method with the actual Rails 8 generator output. Verified via
  WebFetch of the [generator template](https://github.com/rails/rails/blob/main/railties/lib/rails/generators/rails/authentication/templates/app/channels/application_cable/connection.rb.tt):
  the generated `ApplicationCable::Connection#set_current_user`
  uses `Session.find_by(id: cookies.signed[:session_id])`.
- Honest reframing of cross-origin SPA / mobile WebSocket auth:
  the Rails 8 generator does NOT ship a token-based flow.
  Documented three real options (same-origin, CORS+credentials,
  custom token column) with their tradeoffs.

### L36 Encryption: corrected APIs and added missing caveat

- `support_unencrypted_data: true` is a global Rails config
  setting, NOT a per-attribute option as I had written. Verified
  via the [encryption guide](https://guides.rubyonrails.org/active_record_encryption.html).
  Reframed migration procedure as: set the global flag, deploy,
  backfill, remove flag.
- Removed fabricated `attribute_was_encrypted?(...)` method.
  Replaced backfill code with `User.find_each(batch_size: 1000, &:encrypt)`
  using the real `record.encrypt` method from the encryption guide.
- Added critical caveat: rotation is NOT supported for deterministic
  encryption (per encryption guide). The level uses deterministic
  encryption on email; without this caveat, the rotation procedure
  would silently break the equality lookups deterministic encryption
  exists to support.

### Caveat about session-time changes

Claude Code's settings watcher only watches directories that already
had a settings file when the session started. Mid-session edits to
settings.json may not be picked up; the user can open `/hooks` to
force a reload, otherwise the hook activates next session.
