```

                    ^  |
                    |  v
Request -> Router -> Controller -> Response
                    ^  |
                    |  v
                   Model (L5+)
                    ^  |
                    |  v
                  Database (L5+)
```

## Session 18: Redesign L16 Service Objects Visualization

### Problem
L16 used a generic PipelineFlow (Request -> Router -> Controller -> Model -> Database) which is wrong for a refactoring concept. The audit-level skill explicitly says refactoring levels should use "before/after comparisons" or custom zone layouts, not PipelineFlow. PipelineFlow is for request lifecycle concepts (auth, CORS, middleware).

### Solution
Replaced PipelineFlow with a custom "Responsibility Extraction" zone layout:

**Observe phase:**
- A large "Controller" zone (destructive border, "80 lines" badge) containing 4 clickable responsibility blocks: User Creation, Welcome Logging, Default Preferences, Token Generation
- Each block has a pulsing Search icon until inspected. Clicking opens StageInspector showing why that responsibility doesn't belong in the controller
- ProbeTerminal fires 3 registration probes. On probe fire, flow animation highlights each responsibility block sequentially (800ms per block) showing inline execution timing or crash cascading
- Flow messages are concept-specific: "register-slow" shows per-block timing (200ms, 800ms, 1200ms, 1000ms), "register-fails-midway" shows CRASH at logging then "Never reached" for rest
- Probes disabled during flow animation (`disabled={flowPhase !== -1}`)

**Reward phase:**
- Two-zone horizontal layout: thin Controller (8 lines, success border) -> FlowConnector -> Service zone (contains all 4 extracted responsibility blocks)
- Service zone reacts to stress tests: success (green ring) or failure (red ring) with Result display
- Flow animation: controller highlights -> connector animates -> service highlights with result message
- StressTestPanel below with 5 registration scenarios

### What's different from the old version
- Removed all PipelineFlow/PipelineStage/PipelineConnection imports and usage
- Added FlowConnector import for zone-to-zone connections
- Added flow animation system (flowPhase, flowMessages, runFlow) matching L10/L11 pattern
- Separate reward flow system (rewardFlowPhase) for the two-zone reward layout
- Responsibility blocks are button elements with hover/active states, not pipeline nodes
- The visualization now teaches WHY service objects exist (too many responsibilities crammed together) instead of just showing a generic request pipeline
