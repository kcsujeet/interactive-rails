## Session 1: Redesign L15 CORS Visualization

### What Changed

**File:** `frontend/src/features/act2-users-security/components/Level15CORS.tsx`

Redesigned the L15 CORS visualization from a broken layout to an accurate 3-zone horizontal flow.

**Before (problems):**
- Layout: Browser (left) | Origin Boundary (narrow w-20 divider) | Server Tower (right, 3 stacked boxes: Preflight OPTIONS, CORS Middleware, Rails API)
- Preflight was a separate clickable server-side box (it's actually a browser-sent request type)
- CORS Middleware sat between Preflight and Rails API as a middle layer (it's actually the first thing requests hit)
- No flow between the 3 server boxes (stacked vertically, disconnected)
- Single FlowConnector only spanned the narrow origin boundary divider

**After (3-zone horizontal flow):**
```
[Browser/Client] --FC1--> [CORS Middleware Gate] --FC2--> [Rails API]
```

- 3 horizontal zones with 2 FlowConnectors (matches L10 Validations pattern)
- 5 flow phases: Zone 0 highlight -> FC1 animate -> Zone 1 highlight -> FC2 animate -> Zone 2 highlight
- Preflight removed as a zone, communicated through flow messages instead (accurate: it's browser behavior)
- CORS Middleware is now the center gate zone (accurate: first thing requests hit)
- curl probe: Zone 0 switches from browser chrome to terminal style, CORS gate shows "(bypassed)", API shows "200 OK"
- Browser probes: red flow, CORS gate shows blocking, API shows "not reached" with opacity-50
- Stage inspectors reduced from 4 to 3 (browser, cors, api; removed preflight)
- Browser StageInspector updated to explain preflight as part of browser behavior
- Data maps updated: `PROBE_PIPELINE_MAP` uses `gateSublabel`/`gateBadge` instead of `corsSublabel`/`corsBadge`
- `OBSERVE_FLOW` and `REWARD_FLOW` messages updated to match 3-zone semantics

**What stayed the same:**
- 3 probes, 4 discoveries, minRequired: 3
- 6 stress test scenarios
- Build phase (3 steps: terminal + 2 OptionCard)
- Left panel, right panel, code preview
- Flow state machine (runFlow/clearFlow/flowPhase)

## Session 1c: Fix FlowConnector Line Stretching Across Full Width

### What Changed

**File:** `frontend/src/features/act2-users-security/components/Level15CORS.tsx`

All 4 FlowConnector instances had `className="h-4 w-full"` but were missing `relative`. The FlowConnector's static line and dot track use `absolute` positioning, so without `relative` on the container, they positioned relative to the nearest positioned ancestor and stretched across the entire visualization area.

**Fix:** Changed all 4 instances to `className="relative h-4 w-full"`.

## Session 1d: Gate Zone Content Behind Flow Phase

### What Changed

**File:** `frontend/src/features/act2-users-security/components/Level15CORS.tsx`

Zone 1 (CORS gate) and Zone 2 (Rails API) were showing their result text ("No headers sent", "BLOCKED", "not reached", "200 OK") immediately when a probe fired, before the flow animation reached them. This is because `probeState` was derived from `lastProbeId` which updates instantly.

**Fix:** Added two derived flags that gate content display behind the flow reaching each zone:
- `gateRevealed = probeState && (flowPhase >= 2 || flowPhase === -1)` for Zone 1
- `apiRevealed = probeState && (flowPhase >= 4 || flowPhase === -1)` for Zone 2

Replaced all `probeState` references in Zone 1's styling/content with `gateRevealed`, and in Zone 2 with `apiRevealed`. Now zones only show their result state after the animation reaches them (or after it completes).

## Session 1e: Defer Discovery Until Animation Ends

### What Changed

**File:** `frontend/src/features/act2-users-security/components/Level15CORS.tsx`

The "Build the Fix" button was appearing mid-animation because `discoveryGating.discover()` was called immediately in `handleProbe`, which could unlock the button before the flow finished.

**Fix:** Deferred `discoveryGating.discover(discoveryId)` into a `setTimeout` matching the animation end time (`delay * (totalPhases + 2)`). The timeout is pushed into `flowTimeoutsRef` so it gets cleaned up with `clearFlow()`. Now discoveries register only after the animation completes.
