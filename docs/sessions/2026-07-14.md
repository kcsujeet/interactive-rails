## Open-source pivot: fully client-side, no backend (the full change)

The owner decided to open-source the project (MIT) as a free, client-side-only app: all progress in localStorage, no server, no account.

### Backend removed
Deleted `src/server/` (Hono API, repositories, better-auth, D1 schema), `src/pages/api/`, `wrangler.jsonc`, the auth pages/stores/client (`login.astro`, `signup.astro`, `authStore.ts`, `auth-client.ts`, `components/auth/`), the empty `battle/` page, and the unused `IncidentFeed` component. IncidentFeed also held the pure incident-generator helpers the sandbox simulation uses; those were extracted to a React-free `src/lib/incidents.ts` and the import in `usePipelineSimulation.ts` was repointed.

### Progress went local-first (TDD)
`src/lib/progress.ts` rewritten: a pure `mergeCompletion` over an injectable `ProgressStorage`, keep-max stars/score, corrupted-JSON recovery, the legacy storage key kept (`interactive_rails_progress_v1`) so existing players keep progress. 10 new tests written first (red), then the implementation. Call sites rewired: `useLevelCompletion`, `LevelPlayApp` (dropped the server-only `timeToComplete`/`finalMetrics`), `ActsListApp` (removed the guest/account distinction and banner), `DashboardCodebaseButton`. The dashboard was rewritten to render real local data (completed count, stars, chosen database) client-side, dropping the server-only XP/level/title/playtime gamification. The header lost its auth block and now links to Progress.

### Static build
`astro.config.mjs` -> `output: 'static'`, no adapter. The `[actId]/[levelId]` routes gained `getStaticPaths()` (new `getActLevelStaticPaths()` in the registry). The complete page reads its `?stars=N` client-side (static prerender cannot see runtime query strings). `bun run build` now emits 178 static pages to `dist/`, hostable anywhere.

### Deps + config pruned
Removed `@astrojs/cloudflare`, `hono`, `@hono/zod-validator`, `better-auth`, `nanostores`, `@nanostores/react`, `@tanstack/react-query`, `zod` (all unused after backend removal), and `@cloudflare/workers-types`. Dropped the `deploy`/`db:migrate` scripts and the miniflare override. `tsconfig.json` no longer pins workers-types. Removed tracked backend env files (`.env.example` documenting JWT/D1/Workers vars, `.env.development`) and local Cloudflare artifacts; confirmed `.env` was never tracked, so no real secret ever leaked.
