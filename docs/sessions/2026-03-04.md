# Session: 2026-03-04 Session 1

## Summary

Landing page visual overhaul, theme system upgrade, and Button component fix.

## Changes

### 1. Fixed Dark Mode Not Applying

- `BaseLayout.astro` was missing `class="dark"` on `<html>`, so the entire `.dark` CSS theme was never applied
- Page was rendering in light mode with washed-out default shadcn colors
- Added inline `<script>` in `<head>` to read `localStorage('theme')` and apply dark class before paint

### 2. Discovered Radix Slot + Astro SSR Incompatibility

- CTA buttons on landing page were invisible (no background/border)
- Root cause: `@radix-ui/react-slot` does not merge className onto children during Astro's React SSR rendering
- Tested both `@radix-ui/react-slot` and `radix-ui` packages, both failed
- Even `React.cloneElement` approach failed in Astro's renderer
- Works in plain Node SSR but not in Astro's specific React renderer

### 3. Removed asChild/Slot Pattern Entirely

- Simplified `Button.tsx` to always render as `<button>`, removed Radix Slot dependency
- Removed all `asChild` usages across 7 files:
  - `Header.astro`, `login.astro`, `signup.astro`, `dashboard.astro`
  - `SandboxApp.tsx`, `ActsListApp.tsx`, `LevelHeader.tsx`
- Added warnings to `CLAUDE.md` and `MEMORY.md` about this incompatibility

### 4. Theme Color Updates (Three Iterations)

- **Warm brown -> Neutral cool**: Changed dark theme backgrounds from warm hue 30 to neutral hue 260
- **Amber -> Gold**: Shifted primary from dull amber to brighter gold
- **Gold -> Ruby red**: Final theme, primary `oklch(0.65 0.25 18)` dark / `oklch(0.55 0.25 18)` light
- Updated all hardcoded oklch references across `index.astro`, `login.astro`, `signup.astro`, `ActsListApp.tsx`
- Updated favicon colors to ruby red (`#dc2626` spike, `#f87171` accent)
- Updated selection highlight and shimmer animation colors

### 5. Light/Dark Theme Toggle

- Added sun/moon toggle button in Header, outside `#header-auth` div (so auth script innerHTML replacement doesn't destroy it)
- `BaseLayout.astro` inline script reads localStorage before paint to prevent flash
- Defaults to light mode (no class = light)
- Both light and dark mode now use ruby red primary

### 6. Header Height Fix

- Increased header from `h-12` to `h-16` for better proportions

### 7. Hero Section Visual Improvements

- Stronger glows, semi-transparent white borders (`border-white/[0.08]`)
- Better pipeline card depth with layered shadows
- Hero gradient text changed from `via-amber-400` to `via-rose-400`

## Lessons Learned

- Radix Slot is completely broken in Astro's React SSR renderer. Do not use `asChild` in any Astro-rendered component.
- When changing theme colors, there are 5 places to update: `global.css` (light + dark), component hardcoded oklch values, `favicon.svg`, `index.astro` hero gradients, and auth page gradients.
- Theme toggle must be placed outside any container whose innerHTML gets dynamically replaced.

# Session: 2026-03-04 Session 2

## Summary

Fixed 10 light mode visual issues on the landing page. Used cmux browser API for inspection.

## Changes

### 1. Body dot pattern (global.css + BaseLayout.astro)
- Moved dot pattern from CSS `body {}` to Tailwind classes on `<body>` in BaseLayout
- Light mode: black dots (`oklch(0 0 0 / 4%)`), dark mode: white dots (`oklch(1 0 0 / 4%)`)
- CSS variable approach failed (Tailwind v4 tree-shakes unused custom properties)

### 2. Light mode shadow variables (global.css)
- Added `--shadow-sm`, `--shadow-md`, `--shadow-lg`, `--shadow-glow` to `:root`
- Were only defined in `.dark`, causing `.hover-card` to have no shadow in light mode

### 3. Nav button colors (Header.astro + Button.tsx)
- Added `muted` color option to Button: `[--btn:var(--muted-foreground)]`
- Nav ghost buttons use `color="muted"` when inactive (gray text instead of ruby red)

### 4. Hero/CTA radial gradient overlays (index.astro)
- All `oklch(0.65 ... / 0.05)` overlays now have `dark:` variants
- Light mode uses `oklch(0.55 ... / 0.08)` (darker, higher opacity) for visibility

### 5. Terminal header bg (index.astro)
- Changed from `bg-white/[0.03]` (invisible on white) to `bg-zinc-800/50`

### 6. H2 spacing fix (index.astro)
- "Discover Problems<br/>Through Exploration" had no space on mobile (Astro strips whitespace)
- Fixed with space before `<br class="hidden lg:block" />`

### 7. Learning path circles (index.astro)
- Replaced `bg-background` (conflicted with `bg-{color}-500/15`) with `shadow-[0_0_0_4px_var(--background)]`
- Circle text: added `dark:` variants (`text-emerald-600 dark:text-emerald-400`)

### 8. Selection highlight (global.css)
- Light mode: `oklch(0.55 0.25 18 / 20%)` without forced white text
- Dark mode: `oklch(0.65 0.25 18 / 30%)` with white text

### 9. CTA button shadows (index.astro)
- Dual-mode shadows: lighter in light mode, stronger in dark mode

## Architecture Notes

- Tailwind v4 tree-shakes CSS custom properties not referenced by utilities. Can't use CSS vars in raw `body {}` for theme-switching. Use Tailwind classes on the element instead.
- `bg-background` and `bg-{color}/15` conflict in Tailwind v4 since both set `background-color`. Use `shadow-[0_0_0_Npx_var(--background)]` as a masking ring instead.
