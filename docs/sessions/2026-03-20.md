## L35: Fix Code Preview Revealing Answers

### What Changed

**Problem:** The right panel code preview showed the correct answer for OptionCard steps before the player selected it. For example, step 3 ("Add Model Attachment") showed `user.rb` with the correct `has_one_attached :avatar` + variants code while the player was still choosing. This applied to all OptionCard steps (2-5).

**Root cause:** `getCodeFiles` used `furthestStep` to decide which code to show. Each `<= N` boundary showed the RESULT of step N, but it was displayed while the player was still WORKING ON step N. The code preview was always one step ahead.

**Fix:** Rewrote `getCodeFiles` to use `completedStep` (the index of the last completed step, -1 if none). The call site now passes:
```tsx
stepper.isCurrentStepCompleted ? stepper.currentStep : stepper.currentStep - 1
```

This means:
- While working on step N (not completed): show result of step N-1 (context, not the answer)
- After completing step N (before clicking Next): show result of step N (what they just built)

Also added a new code preview for `completedStep === 1` (after running migrations): shows `db/schema.rb` with the created tables + an unconfigured `config/storage.yml` hint. Previously this state didn't exist because the boundaries were shifted.

- `bun test`: All 65 L35 tests pass (300 assertions)

## L35: Fix Direct Upload Animation Using Wrong Connector

### What Changed

**Problem:** The reward phase direct upload animation used `connB` (App Server <-> S3) to show the Client uploading to S3. But `connB` is the connector between App Server and S3 in the linear layout `Client -- connA -- App Server -- connB -- S3`. Direct upload means the file goes Client -> S3 directly, bypassing the App Server. Using `connB` contradicts the core lesson.

**Fix:** Added `connC` (Client <-> S3 direct) as a new connector that bypasses the App Server.

- Added `connC` to `AnimationFrame` interface
- Added `connCState` state variable, wired into `resetViz` and `applyFrame`
- Updated 3 reward frame arrays to use `connC` instead of `connB`:
  - `REWARD_DIRECT_UPLOAD` frame 3: Client -> S3 direct
  - `REWARD_CONCURRENT` frame 3: 10 clients -> S3 direct
  - `REWARD_SERVE_VARIANT` frame 3: S3 -> Client direct (CDN redirect)
- `REWARD_ATTACH_VARIANT` correctly keeps `connB` (App Server generates variant, sends to S3)
- Initially rendered connC as an SVG arc overlay, but replaced with a full React Flow implementation (see below).

**Audit skill and testing rules updated:**

- **SKILL.md visualization accuracy checklist:** Added "Connectors accurately represent the data path for each animation" check. If a scenario bypasses a zone, the animation must use a bypass connector, not the through-zone connector. Includes case study and examples (CDN redirects, webhook callbacks, background jobs).
- **testing.md:** Added "Animation connector accuracy" test category for verifying bypass scenarios use the correct connector.

## L35: Replace Custom Visualization with React Flow

**Problem:** The custom div-based zone layout with FlowConnector components couldn't properly render the Client <-> S3 direct arc. The SVG overlay approach was disconnected from the nodes.

**Fix:** Replaced the entire `renderUploadPipeline` with a React Flow instance using custom node and edge types.

- **`ZoneNode`:** Renders the full zone content (icon, title, dynamic label with flash colors, MemoryGauge, Local Disk indicator). Uses `Handle` components for edge connections (right/left for horizontal, top for the arc).
- **`UploadEdge`:** Renders edges with animated `animateMotion` dots. Uses `getSmoothStepPath` for horizontal edges and `getBezierPath` with `curvature: 0.8` for the direct arc (connC). `isDirect` flag controls dashed stroke and "Direct:" label.
- **Layout:** 3 nodes at fixed coordinates. Observe: 2 nodes (Client, App). Reward: 3 nodes + connC arc via top handles.
- **Removed:** `DirectArc` component, `FlowConnector` import, all manual div-based zone/connector rendering.
