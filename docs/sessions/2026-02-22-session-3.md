## Level 13: Replace SVG Pipeline with React Flow Visualization

### Changes Made

**File modified:** `frontend/src/features/act2-users-security/components/Level13Testing.tsx`

### What Changed

Replaced the hand-crafted SVG `DeployPipeline` component with a pure React Flow solution (`DeployPipelineFlow`) using `@xyflow/react` v12.4.3.

**Added:**
- `PipelineStageNode` (memo'd custom node): Styled div with invisible Handles, shows label/sublabel/badge. Variants: `default` (dark zinc) and `test` (inactive = dashed/dimmed, active = emerald solid)
- `PipelineFlowEdge` (custom edge): Uses `getSmoothStepPath` + `<BaseEdge>` with `<animateMotion>` circles for flowing dots
- `MIXED_DOTS` config: 4 dots per edge (3 green, 1 red), staggered with negative `begin` values
- `CLEAN_DOTS` config: 3 green-only dots for test->prod edge in reward phase
- `DeployPipelineFlow`: Stateless wrapper, receives `testGateActive` boolean, uses `useMemo` for nodes/edges
- `pipelineNodeTypes` and `pipelineEdgeTypes` defined outside component (stable references)
- Simple counter interval for reward phase (increments caught/deployed every 3.5s)

**Removed:**
- `DeployPipeline` SVG component (the entire hand-crafted SVG with manual coordinates)
- `PipelineCommit` interface
- `commits` state, `animationActive` derived value, `testGateActiveRef` ref
- Two animation `useEffect`s (spawn interval at 900ms + move interval at 50ms with position tracking)
- `FlaskConical` unused import, `useRef` import

**Unchanged:**
- Phase state machine (observe -> build -> activate -> reward)
- Build phase (6 steps: 3 terminal + 3 OptionCard)
- `getCodeFiles()`, `PipelineLegend`, left/right panel content
- `content.ts`

### Technical Decisions

- Followed the React Flow `AnimatedSVGEdge` pattern from docs: `<BaseEdge>` + `<circle>` with `<animateMotion>`
- Used `getSmoothStepPath` for smooth step edges (horizontal flow between nodes)
- Disabled all interactivity on ReactFlow (dragging, zooming, panning, selection) since it's a visualization-only component
- Used `fitView` with `padding: 0.4` for responsive sizing
- Node positions: x=0, 250, 500, 750 (evenly spaced horizontally at y=0)
- Dots use negative `begin` values so they appear pre-distributed on mount instead of all starting from the same position

### Verification

- `bun run build` passes with no TypeScript errors

## Level 14: Rebuild as CORS & Credentials (Phase 1 of Security Split)

### Context

Level 14 previously covered 4 independent topics (CORS, credentials, rate limiting, strong params) in a legacy "click findings" pattern with no phases. Splitting into two focused levels:
- Level 14: CORS & Credentials (this change)
- New Level 15: Rate Limiting + Strong Params (Phase 2, separate session)

### Files Modified

- `frontend/src/features/act2-users-security/components/Level14Security.tsx` (complete rewrite)
- `frontend/src/features/act2-users-security/content.ts` (Level 14 content updated)

### What Changed

**Component rewrite (Level14Security.tsx):**

Complete rewrite from legacy "click findings" pattern to canonical three-phase sequential flow (matching Level 12 reference implementation).

- Phase state machine: observe -> build -> activate -> reward
- 4 build steps: 2 terminal (bundle add rack-cors, EDITOR=vim rails credentials:edit) + 2 OptionCard (configure CORS origins, access credentials in code)
- Custom `SecurityShieldVisualization` SVG component (viewBox 600x400):
  - Central "Rails API" box with CORS shield (left) and Credentials shield (right)
  - Observe mode: dashed shields with "!" badges, red dots pass through
  - Reward mode: solid emerald shields with checkmarks, red dots fade with opacity animation, "BLOCKED" text pulses
  - Pure SVG with `<animateMotion>` and `<animate>` elements
- `ShieldLegend` component (Shield icon = active protection, X icon = threat)
- `getCodeFiles()` evolves through 5 stages (vulnerable -> Gemfile -> cors.rb -> credentials.yml.enc -> stripe.rb with credentials.dig)
- Single `blockedCount` counter in reward phase (incremented every 3500ms)
- Star rating flows through `stepper.starRating`

**Removed from old component:**
- `SecurityFinding` interface and `INITIAL_FINDINGS` array (4-finding click pattern)
- `ICON_MAP`, `handleFixOption`, `selectFinding` handlers
- Grid-based findings overview UI
- `useLevelCompletion` hook (replaced by `useStepGating` + `onComplete`)

**Content changes (content.ts):**
- `name`: "Security" -> "CORS & Credentials"
- Removed: `startingPipeline`, `successConditions`, `availableNodes`, `unlockedNodes`
- Narrowed `trigger.description`, `problem.observation`, `problem.codeExample`, `problem.goal` to CORS + credentials only
- Removed rate_limit and strong_params sections from `learningContent`
- Updated `hint.text` with step-by-step guidance
- Removed "Rails 8 Rate Limiting" from `furtherReading`

### Technical Decisions

- Used pure SVG (not PipelineFlow) because the shield/firewall concept requires a different layout than a linear pipeline
- Motion paths defined with `<path>` elements in `<defs>` for `<animateMotion>` targeting
- Shield shapes use SVG `<path>` with quadratic curves for the classic shield silhouette
- Protected mode uses opacity animation on dots to show them fading at shields
- Step data uses the same `TerminalChoiceStep` + `OptionCard` pattern as Level 12

### Verification

- `bun run build` passes with no TypeScript errors
- All 4 phases present (observe -> build -> activate -> reward)
- 4 build steps work with correct feedback messages
- Code preview evolves through all stages
