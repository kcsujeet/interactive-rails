# Session: 2026-02-22-session-3

## Level 13: Replace SVG Pipeline with React Flow Visualization

### Changes Made

**File modified:** `frontend/src/features/act2-users-security/components/Level13Testing.tsx`

### What Changed

Replaced the hand-crafted SVG `DeployPipeline` component with a pure React Flow solution (`DeployPipelineFlow`) using `@xyflow/react` v12.4.3.

**Added:**
- `PipelineStageNode` (memo'd custom node): Styled div with invisible Handles, shows label/sublabel/badge. Variants: `default` (dark zinc) and `test` (inactive = dashed/dimmed, active = emerald solid)
- `PipelineFlowEdge` (custom edge): Uses `getSmoothStepPath` + `<BaseEdge>` with `<animateMotion>` circles for flowing dots
- `MIXED_DOTS` config: 4 dots per edge (3 green, 1 red), staggered with negative `begin` values
- `CLEAN_DOTS` config: 3 green-only dots for test->prod edge in reward phase
- `DeployPipelineFlow`: Stateless wrapper, receives `testGateActive` boolean, uses `useMemo` for nodes/edges
- `pipelineNodeTypes` and `pipelineEdgeTypes` defined outside component (stable references)
- Simple counter interval for reward phase (increments caught/deployed every 3.5s)

**Removed:**
- `DeployPipeline` SVG component (the entire hand-crafted SVG with manual coordinates)
- `PipelineCommit` interface
- `commits` state, `animationActive` derived value, `testGateActiveRef` ref
- Two animation `useEffect`s (spawn interval at 900ms + move interval at 50ms with position tracking)
- `FlaskConical` unused import, `useRef` import

**Unchanged:**
- Phase state machine (observe -> build -> activate -> reward)
- Build phase (6 steps: 3 terminal + 3 OptionCard)
- `getCodeFiles()`, `PipelineLegend`, left/right panel content
- `content.ts`

### Technical Decisions

- Followed the React Flow `AnimatedSVGEdge` pattern from docs: `<BaseEdge>` + `<circle>` with `<animateMotion>`
- Used `getSmoothStepPath` for smooth step edges (horizontal flow between nodes)
- Disabled all interactivity on ReactFlow (dragging, zooming, panning, selection) since it's a visualization-only component
- Used `fitView` with `padding: 0.4` for responsive sizing
- Node positions: x=0, 250, 500, 750 (evenly spaced horizontally at y=0)
- Dots use negative `begin` values so they appear pre-distributed on mount instead of all starting from the same position

### Verification

- `bun run build` passes with no TypeScript errors

## Extract Reusable PipelineFlow Component

### Changes Made

**Created:** `frontend/src/components/levels/PipelineFlow.tsx`
**Modified:** `frontend/src/components/levels/index.ts`, `frontend/src/features/act2-users-security/components/Level13Testing.tsx`

### What Changed

Extracted the inline React Flow pipeline visualization (~240 lines) from Level 13 into a reusable `PipelineFlow` component in `components/levels/`.

**New `PipelineFlow.tsx`:**
- `PipelineStageNode` (memo'd): Variants changed from `'default' | 'test'` + `active` boolean to `'default' | 'active' | 'inactive'` (simpler API)
- `PipelineFlowEdge`: Custom edge with `<animateMotion>` dots (same logic)
- Exported presets: `PIPELINE_DOTS_MIXED` (4 dots: 3 green + 1 red) and `PIPELINE_DOTS_CLEAN` (3 green)
- Auto-layout: stages positioned 250px apart horizontally, no manual positions needed
- `PipelineConnection.dots` accepts `PipelineDot[]`, `'mixed'`, `'clean'`, or undefined
- Dot preset strings resolved internally with unique IDs per edge

**Updated `index.ts`:**
- Exports types: `PipelineConnection`, `PipelineDot`, `PipelineFlowProps`, `PipelineStage`
- Exports values: `PIPELINE_DOTS_CLEAN`, `PIPELINE_DOTS_MIXED`, `PipelineFlow`

**Updated Level 13:**
- Removed: All inline React Flow code (DotConfig, StageNodeData, PipelineStageNode, PipelineFlowEdge, pipelineNodeTypes, pipelineEdgeTypes, makeStageNodes, makeEdges, DeployPipelineFlow, MIXED_DOTS, CLEAN_DOTS)
- Removed imports: `@xyflow/react`, `@xyflow/react/dist/style.css`, `memo`, `useMemo`, `cn`
- Added: Two declarative stage/connection configs (`OBSERVE_STAGES/CONNECTIONS`, `REWARD_STAGES/CONNECTIONS`) and `<PipelineFlow>` usage in observe + reward phases

### Net effect

Level 13 lost ~240 lines of inline React Flow code and gained ~30 lines of declarative config. `PipelineFlow` is now importable from `@/components/levels` for any future level needing a "stages with flowing dots" visualization.

### Verification

- `bun run build` passes with no TypeScript errors
