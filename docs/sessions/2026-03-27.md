## Level 39 Webhooks & Idempotency - Full Redesign

### Changes

**Removed activate phase.** Build phase now transitions directly to reward on last step's "Next Step" button. No star rating, no "Visualize Webhook Security" interstitial.

**Replaced static gate boxes with React Flow 4-node visualization.** Old design used vertical stacked div boxes with FlowConnector (wrong props: `direction="down"`, `variant`). New design uses FlowDiagram with 4 custom React Flow nodes:
- Stripe/Attacker node (left) with dynamic icon (Globe vs ShieldAlert)
- App Server node (center-left) with label + badge
- Database node (upper-right) with mini-table rows
- Job Queue node (lower-right) with status label

Three edges with AnimatedDots, edge labels, and reversePath for response flows.

**Rewrote probes as user stories.**
- "POST webhook (forged, no signature)" -> "Attacker forges payment webhook"
- "POST webhook (duplicate event_id)" -> "Stripe retries payment event (network hiccup)"
- "POST webhook (slow handler)" -> "Monthly invoice webhook (slow handler)"
- Added `story: string[]` fields with 4-6 bullet points each

**Added multi-frame animations.** Each probe has 4-7 animation frames showing the mechanism step by step: dots flowing between nodes, labels updating, colors changing. Animation speed scaled to 2x (4 nodes + 3 edges), with duplicate-event probe at 2.5x due to having the most frames.

**Rewrote stress scenarios to replay observe probes.** Old scenarios were feature demos (generic "POST payment.succeeded (valid)"). New scenarios are paired with observe probes:
- "Attacker forges payment webhook (with verification)" - same attacker, rejected at signature
- "Stripe retries payment event (with dedup)" - same retry, caught by dedup check
- "Monthly invoice webhook (with async)" - same slow invoice, processed in <50ms
- Plus 2 reward-only scenarios (valid-subscription, bad-payload)
- Added `responseLines` and `story` fields to all scenarios

**Fixed code preview index.** Changed from `furthestStep` to `isCurrentStepCompleted ? currentStep : currentStep - 1`.

**Fixed ErrorFeedback placement.** Moved above options, not below.

**Fixed "Next Step" button.** Changed to `className="gap-2" size="sm"` matching standard.

**Fixed StressTestPanel props.** Changed from passing `stressTest` object to individual props (results, allowedCount, blockedCount, etc.) matching the actual interface.

**Fixed DiscoveryChecklist props.** Changed from custom `{found: boolean}` mapping to `discoveryGating.discoveries` with proper `status` field.

**Fixed StepProgress props.** Changed from `STEP_DEFS.map(s => s.title)` strings to `stepper.steps` with proper `Step` objects.

**Fixed OptionCard props.** Changed from nonexistent `code`/`label`/`correct`/`onSelect` props to actual API: `name`/`mono`/`disabled`/`selected`/`onClick`.

**Added LevelHeader.** Custom level components must include their own `LevelHeader` inside `CenterPanel`. Added with `onComplete`, `onReset`, and `onValidate` handlers (requires 3+ stress test fires to submit).
