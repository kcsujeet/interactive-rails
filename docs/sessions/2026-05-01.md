## Phase 4: Encode pedagogy rules into lints

Goal: move the most-violated rules from "things authors must remember" to "things the test suite catches".

### New shared helpers — `src/lib/testing/level-pedagogy.ts`

Each helper enforces a rule from `.agents/rules/pedagogy.md` or `level-content.md`:

| Helper | Rule |
|---|---|
| `expectAllDiscoveriesRequired` | minRequired must equal DISCOVERY_DEFS.length. |
| `expectProbeDiscoveryMapOneToOne` | Each probe unlocks exactly one distinct discovery; each discovery is unlocked by exactly one probe. |
| `expectStoriesPresent` | Every probe and scenario has 3-6 substantive story bullets (each ≥ 20 chars). |
| `expectProbesMatchScenarios` | Probe-to-scenario coverage with same id and label. |
| `expectBuildStepQuality` | Exactly one correct option, never at index 0, unique ids/labels, substantive wrong-option feedback that does not quote the correct label or any answer-keyword. |
| `expectScenarioBasics` | Unique scenario ids and labels, mix of allowed/blocked results. |

Each helper aggregates errors across the whole input set and fails with one assertion that lists every issue, so a failing test reports all problems at once instead of one at a time.

### Curriculum-wide checks added to `scripts/validate-levels.ts`

- Slug shape `act{N}-level{M}-{kebab-name}` parsed and cross-checked against `actId` / `levelNumber`. Catches the renumbering bug class where a directory was moved but the slug inside content was not updated.
- Level numbers must be contiguous L1..L58 with no gaps or duplicates.
- Existing required-field checks preserved.

### Reference implementation: Level49Deployment.test.ts

Refactored to demonstrate the helper pattern: one `describe('shared pedagogy lints')` block calls the helpers; one `describe('level-specific shape')` block keeps the per-level assertions (exact counts, exact ids, theme-word matching for paired probe/scenario labels) that no shared helper can know about.

Test file: 301 -> 150 lines with stronger assertions. The level-specific block notes when a generic helper is intentionally skipped (L49 uses a `PROBE_TO_SCENARIO` map with different ids, so `expectProbesMatchScenarios` is replaced by a custom thematic-word test).

### Documentation

`.agents/rules/pedagogy.md` "Test enforcement" section rewritten with a table mapping each helper to the rule it enforces. Authors land on this section and immediately see which rules are mechanically caught vs which still rely on careful authoring.
