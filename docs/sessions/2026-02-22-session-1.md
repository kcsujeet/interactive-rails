## Lessons Learned

- The codebase had THREE different slug patterns: `railsexpert` (one word), `rails-expert` (hyphenated), and `Rails Expert` (spaced). Each needed separate grep patterns to find.
- `bun install` does not always update `bun.lock` name fields when `package.json` names change. Deleting `bun.lock` and reinstalling forces a clean regeneration.
- The Edit tool requires reading a file in the current conversation context before editing, even if a subagent already read it.
- Don't use `TerminalChoiceStep` for non-shell-command choices. Ruby code snippets (like `render json:`) are not terminal commands and shouldn't appear with a `$` prompt.
- `useLevelCompletion().completeLevel()` has no guest fallback. The parent's `completeLevelProgress()` does. Don't gate `onComplete` on the level-internal API call.
- `bundle add` is not the conventional Rails workflow. The standard pattern is: edit Gemfile, then `bundle install`.
- Parallelizing P2 enhancements as 11 independent agents (3 batches of 4+4+3) works well since each modifies a different file with no conflicts.
- Phase gating (L16) and data leak visualization (L49) turn passive levels into active problem-solving exercises.
