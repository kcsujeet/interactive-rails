## Level 28 Pagination: Cumulative Pattern Compliance

### Problem
All code previews in L28 showed raw controller code (`Post.includes(:user).all`, `pagy(:offset, Post.includes(:user))`), violating the L16+ service object pattern and L18+ dry-validation contract pattern. The zone inspector, `getCodeFiles`, step options, and content.ts all had this issue.

### Changes Made

#### 1. Component (Level28Pagination.tsx)
- **ZONE_INSPECTOR_MAP**: Now shows `PostList < ApplicationService` service code with `ListContract` validation, not raw controller code
- **getCodeFiles (observe phase)**: Shows 3 files: `app/contracts/list_contract.rb`, `app/services/post_list.rb` (with LIKE query highlighted), `app/controllers/api/v1/posts_controller.rb` (thin, delegates to service)
- **getCodeFiles (build step 0)**: Shows the broken service code
- **getCodeFiles (steps 4-5)**: Controller now calls `PostList.call(page: params[:page])` and paginates `result.scope`
- **getCodeFiles (reward)**: Shows contract + service alongside the controller
- **WIRE_INDEX_OPTIONS**: Changed from `Post.includes(:user)` to `result.scope` (service returns the scope, controller paginates it)
- **OPTION_STEP_CONFIG step 3 description**: References "result.scope" from the service

#### 2. Content (content.ts)
- **problem.observation**: References "PostList service" instead of "controller"
- **problem.codeExample**: Shows service object code with `ListContract` validation
- **railsCodeExample**: Shows full service + controller pattern with `PostList.call` and `pagy(:offset, result.scope)`

#### 3. Test File
- Updated WIRE_INDEX_OPTIONS to use `result.scope` references
- All 63 tests pass

### Architecture Decision
Pagy's `pagy()` method is inherently a controller concern (reads `params[:page]`, returns pagination metadata). The service provides the scope; the controller paginates it. This is the correct separation: service handles business logic (what to query), controller handles HTTP concerns (pagination, headers).

## Level 29 Search: Grid and Text Size Polish

### Problem
The document grid blocks were tiny (`w-2.5 h-2.5` = 10px) with a fixed `maxWidth` of 260px, not filling the parent container. All text used `text-[10px]` (10px) which was too small.

### Changes
- **Grid**: Replaced `flex flex-wrap` with fixed maxWidth with `grid grid-cols-[repeat(20,1fr)] gap-1 w-full` so blocks fill the full width
- **Block size**: `w-2.5 h-2.5` -> `h-4` (16px height, width auto from grid)
- **All `text-[10px]`** -> `text-xs` (12px) across DocumentGrid, GinIndexCard, NoIndexBanner, BlockedBanner, SearchBadges, and zone content
- **`text-[11px]`** -> `text-xs` on GinIndexCard title and banners
- **Legend swatches**: `w-2 h-2` -> `w-2.5 h-2.5` (inline grid legend), `w-3 h-3` (reward left panel legend)
- **Removed unused `GRID_COLS`** constant (was only used for the old maxWidth calc)
