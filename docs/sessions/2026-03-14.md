## StressTestPanel: Fix Unbounded Growth

### Problem
When `className="flex-1 flex flex-col"` is passed, the results log area grows unbounded as results are added, pushing the pipeline diagram off screen. ProbeTerminal handles this correctly by using `flex-1 min-h-0` which constrains within the flex parent and enables scrolling.

### Fix (StressTestPanel.tsx)
Two layout patterns established based on visualization type:

**Pattern A (custom viz, e.g. L30):** Visualization has natural height. Terminal fills remaining space via `className="flex-1 flex flex-col"`. Wrapper uses `flex-1 min-h-0 flex flex-col`. Results log: `flex-1 min-h-0 overflow-y-auto`.

**Pattern B (PipelineFlow, e.g. L31):** Pipeline uses `flex-1` to fill space. Terminal has bounded fixed size, NO `className` prop. Plain wrapper. Results log: `min-h-36 max-h-64 overflow-y-auto`.

- **Results log**: Conditional `className ? 'flex-1 min-h-0' : 'min-h-36 max-h-64'`
- **Header/footer**: Added `shrink-0` so they never compress
- **L31 observe/reward**: Reverted to plain wrappers, no className on ProbeTerminal or StressTestPanel
- **L30**: Keeps flex-fill pattern with className
- **Audit skill**: Updated with both patterns and decision criteria
