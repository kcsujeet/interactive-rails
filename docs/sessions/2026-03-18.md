#### Key Design Decisions
- **Numbers changed**: Request A buys 10, Request B buys 8 (instead of both buying 10). This makes the overwrite visually obvious: stock goes 15 -> 5 -> 7 (different values), not 15 -> 5 -> 5
- **Mismatch counter**: "18 units sold, only 8 deducted" makes the consequence concrete
- **Stale edit probe**: Table switches to price columns (id, name, price, lock_version) for the admin edit scenario
- **Reward phase**: Shows lock_version incrementing (0 -> 1 -> 2), Request B waiting then reading fresh data
