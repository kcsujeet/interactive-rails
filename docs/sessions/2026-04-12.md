## Architecture Notes

### Codebase Registry: Zero-Duplication Pattern
Each level calls `registerLevelCode('id', () => getCodeFiles('reward', STEP_DEFS.length))` at module level. The lazy getter reuses the existing `getCodeFiles` function (single source of truth). The registry evaluates lazily when the viewer opens, merging files in level order with last-writer-wins for duplicate filenames.

### Miniflare Pin
`miniflare` is pinned to `4.20251217.0` via `overrides` in package.json due to [workers-sdk#13013](https://github.com/cloudflare/workers-sdk/issues/13013). POST requests returning non-2xx status codes crash newer miniflare versions. Remove the pin once the upstream fix ships.

## Session 2: Content File Splitting (Acts 1-4)

### Split content.ts into per-level files

Split the monolithic `content.ts` file in Acts 1-4 into individual per-level content files and an act metadata file.

**Pattern for each act:**
- `content/content.ts` (deleted) -> `content/act.ts` + `content/level-N-name.ts` (one per level)
- Each per-level file exports a single `Level` object with its own type import
- `act.ts` imports all level files and exports the act aggregate
