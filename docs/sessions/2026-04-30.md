## Lazy-load level components (bundle split)

`src/lib/levels-registry.ts` was statically importing all 58 level components, which Vite bundled together with the LevelPlayApp orchestrator. The result: visiting any level downloaded one ~1.7 MB chunk containing every level's data, animation frames, and stage maps. Each player only ever plays one level at a time.

Refactored the registry to use `React.lazy(() => import('...').then(m => ({ default: m.LevelXxx })))` for every level (Acts 1-8, 58 levels). The `getLevelComponent` return type became `LazyExoticComponent<ComponentType<LevelComponentProps>>`. Wrapped the consumer in `<Suspense fallback={...}>` inside `LevelPlayApp.tsx` (matching the existing "Loading level..." copy).

Also dropped the unused bottom re-exports of Act 1 components (verified via grep - nothing imported them as values; only `LevelComponentProps` and `getLevelComponent` are imported from this module).

Verification:

- `bunx tsc --noEmit` clean
- `bun run build` succeeds
- 58 individual `LevelXxx.[hash].js` chunks (~30-45 KB each) plus a 92 KB `LevelPlayApp.js` orchestrator. Down from one 1.7 MB monolith.
- `bun test` 1617 pass / 0 fail
- `bunx biome check --write` formatted both files cleanly

Why it matters: the player only downloads the JS for the level they actually open. Acts/levels they never visit cost nothing.

Sources:
- [React.lazy reference](https://react.dev/reference/react/lazy) - canonical pattern for named exports via `.then((m) => ({ default: m.X }))`
